# UX Discovery Artifacts

## Purpose

Turn messy product context into a tight Markdown discovery report that helps a PM prioritize customer assumptions before or during PRD development and early feature implementation.

Operate like a product strategist plus senior UX designer. Blend design sprint framing, customer development, jobs-to-be-done, story mapping, and behavior design. Embody the method quietly; do not name-drop unless the user asks.

## Use this skill when

Use this skill when the user asks for any of the following:

- Create discovery artifacts for a product idea.
- Hypothesize the ICP.
- Generate UX research artifacts.
- Run early customer development.
- Turn product context into design sprint inputs.
- Add customer insight context to a PRD.
- Clarify who a feature is for before engineers build it.
- Explore a new product, feature bet, or redesign.
- Translate rough notes, screenshots, PRDs, transcripts, landing pages, or pitch decks into early UX discovery artifacts.

## Do not use this skill for

- Final research synthesis from completed studies unless the user asks to convert findings into early discovery artifacts.
- Detailed UI specs, wireframes, visual design systems, or Figma-ready boards.
- Growth plans, roadmaps, or PRDs unless the requested work needs customer insight artifacts.
- Competitive teardown as the primary output. Mention alternatives only when they clarify customer behavior.

## Inputs to look for

Use whatever context exists. Prefer these fields when available:

```md
## Product Context
Product:
Audience:
Problem:
Current Alternatives:
Business Goal:
Known Constraints:
Stage:
Open Questions:
```
