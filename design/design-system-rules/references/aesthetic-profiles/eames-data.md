# Eames meets data

Optional aesthetic profile. Load this only when the product needs a warm, editorial data interface with strong mid-century character. The universal `references/source.md` checklist still applies.

## Point of view

Industrial optimism, considered utility, and visual confidence. The interface should feel designed like a well-made object, not assembled from generic dashboard parts.

## Expression

- **Ground:** warm paper/cream canvas, near-black ink, restrained stroke grid.
- **Color:** reserve high-chroma color for navigation, selected states, chart series, and meaningful status.
- **Geometry:** precise data grid with rounded color blocks; use a small radius family such as 0/8/16/24px, 1–2px ink strokes, and restrained elevation.
- **Composition:** strong left rail, clear modular columns, controlled asymmetry, and one primary artifact per state.
- **Typography:** bold, high-x-height sans for titles plus disciplined mono for metrics, labels, ranks, and source/version metadata.
- **Density:** skinny toolbars, compact context ribbons, dense tables, and progressive disclosure for secondary methodology.
- **Interaction:** selected states use at least two channels such as fill + border, fill + index, or color + text. Preserve 44px touch targets and visible focus rings on every color block.

## Data visualization

- Use warm paper/ink chart surfaces.
- Show zero baselines for bars.
- Prefer direct labels where possible.
- Use a deterministic saturated series palette separate from semantic status colors.
- Keep gridlines subordinate to marks.
- Pair hue with position, text, or shape.

## Avoid

Dark sci-fi defaults, glassmorphism, gradient backgrounds, excessive shadows, rainbow palettes, pill-everything UI, and decorative color that competes with quantitative evidence.
